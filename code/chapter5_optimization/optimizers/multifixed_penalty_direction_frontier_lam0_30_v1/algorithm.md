# Multi-Fixed-Step Penalty Variant

## Intuition

This optimizer keeps the fixed step-size menu and beam structure of the multi-fixed-step gradient method, but adds a proximity penalty with `lambda=0.30`. Variant mechanics: penalized direction, penalized frontier ranking.

## Configuration

| Parameter | Value |
|-----------|-------|
| Step sizes | `0.005, 0.01, 0.02, 0.05, 0.1, 0.2, 0.5, 1.0, 2.0, 5.0` |
| Max depth | 20 expansion depths |
| Beam width | 12 candidates after expansion |
| Max evaluations | no fixed cap; bounded by depth, beam width, and bracketing |
| Proximity weight | `0.30` |
| Penalized direction | `True` |
| Penalized frontier | `True` |
| Sparse mode | `None` |

## Mathematical Description

When the penalized direction is enabled, the fixed-step probes use

$$
d_t = \frac{\nabla_\Delta P(y=1\mid x_t) - \lambda (x_t-x_0)/\sigma}{\|\nabla_\Delta P(y=1\mid x_t) - \lambda (x_t-x_0)/\sigma\|_2}.
$$

For each configured normalized step size `s`, the candidate is

$$
x_{t+1} = x_t + s\,\sigma\odot d_t.
$$

When the penalized frontier is enabled, valid frontier candidates are ranked by progress in

$$
J_\lambda(x)=P(y=1\mid x)-\frac{\lambda}{2}\left\|(x-x_0)/\sigma\right\|_2^2.
$$

When a fixed-menu candidate crosses the threshold, the segment between its parent
and the crossing candidate is bracket-refined by bisection before final
selection. The selected final design is the nearest valid threshold crossing
unless a sparse mode changes the final selection rule. Ground-truth labels are
used only for final reporting.

## Inputs And Outputs

- Input: one blocked benchmark start, the trained MLP surrogate, benchmark
  normalization statistics, threshold `tau`, and optional parameter-lock
  constraints.
- Output: one selected parameter vector, the surrogate trajectory, the selected
  threshold-crossing candidate if one exists, and post-hoc verification metrics.

## Thesis Interpretation

This variant belongs to the penalty-weight sweep for the thesis-facing
multi-fixed-step penalty method. It keeps the fixed step menu, uses the proximity
penalty in both direction and beam ranking, and bracket-refines threshold
crossings before final selection.

## Failure Modes

- The penalty can make the beam conservative and reduce verified repairs.
- Direction normalization can damp the visible effect of small penalty weights.
- Sparse final selection can only choose among candidates already generated by the fixed-step beam.
